# 🚗 車両詳細ページ 運用・管理マニュアル

## 1. ファイルの命名規則
新規追加時や、同じ車を再販する時の重複を防ぐため、以下のルールでファイルを作成します。

**`car-detail-[車種]-[車台番号下4桁]-[仕入年月].html`** （例：`car-detail-tanto-5678-202605.html`）

> [!TIP]
> **💡 GA4分析のメリット** > この命名規則にしておくことで、GA4（Googleアナリティクス）のイベント計測において、「どの車種のページからLINEや電話のタップが発生したか」がファイル名（URL）として正確に送信され、効果測定が非常に簡単になります。

---

## 2. 新規車両を追加する手順

1. **ファイルの作成** `car-detail-template.html`（テンプレート）をコピーし、上記の命名規則で新しいファイルを作成します。
   
   > **【時短テクニック】** > 生成AI（GeminiやChatGPTなど）に「車検証の情報」や「オークション出品表」、「アピールしたい点」を箇条書きで渡し、*「この情報をもとに、車両詳細ページのHTML（スペック表とおすすめポイント）を出力して」*と指示を出します。出力結果を新しいファイルにコピペすることで大幅な時短になります。

2. **内容の書き換え** 新しいファイル内の「写真・価格」などを正確な情報に書き換えます。

3. **在庫一覧への追加** `cars.html`（在庫一覧ページ）を開き、新しい車両のカード（枠）のHTMLブロックを追加します。

4. **リンクの設定** `cars.html`内の「詳細を見る」ボタンのリンクを、新しく作ったファイル名に設定します。
   ```html
   <a href="car-detail-tanto-5678-202605.html" data-ga-event="click_cars_list_detail" class="...">
   ```

5. **反映** GitHubでセーブ（Commit & Push）します。

## 3. 車両が売れた場合（SOLD OUT表示にする手順） ★推奨
お店の活気をアピールし、また検索エンジンやブックマークからのアクセス（SEO）の取りこぼしを防ぐため、すぐには消さずに「売約済み」として一定期間残します。

### ① 在庫一覧（`cars.html`）の更新
1. `cars.html` を開く。
2. 売れた車両の画像部分に**「SOLD OUT」のラベル**を追記します。
3. 「詳細を見る」「LINEで問合せ」ボタンのブロックを、以下のような**クリックできないボタン**に書き換えます。
   ```html
   <div class="flex-grow bg-gray-400 text-white py-3 rounded-lg font-bold text-center">SOLD OUT（売約済み）</div>
   ```
   ※これにより、一覧ページから詳細ページへのリンクが外れます。

### ② 詳細ページ（`car-detail-***.html`）の更新 ※直接アクセス対策
ブックマークやGoogle検索から直接詳細ページにやってきたお客様を逃さないための処理です。

1. 該当の詳細ファイルを開く。
2. ページ下部の**超強力CTAコンポーネント**（「1台限りの早い者勝ち」のLINE/電話ボタン）のHTMLを削除、またはコメントアウトします。
3. 代わりに**「この車両は売約済みとなりました。似た条件のお車をお探ししますので、お気軽にご相談ください」**というテキストと、総合窓口へのLINEボタン等に差し替えます。
4. GitHubでセーブ（Commit & Push）します。

---

## 4. 車両一覧から完全に削除する手順（販売から1〜2ヶ月経過後）

1. **一覧からの削除** `cars.html` を開き、該当車両のHTMLコードのブロック全体を削除、またはコメントアウト（`` で囲む）して非表示にします。

2. **ファイルの削除** 左側のファイル一覧から、該当する詳細ファイル（例：`car-detail-tanto-5678-202605.html`）を削除します。

3. **反映** GitHubでセーブ（Commit & Push）します。

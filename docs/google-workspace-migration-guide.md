# ドメイン移管およびGoogle Workspace導入 完全ステップガイド

本マニュアルは、現在Jimdoで管理しているドメイン（`carpath.net`）を外部に移管し、有料のGoogle Workspace（旧G Suite）で自社ドメインのGmailを利用できるようにするための具体的な手順書です。

専門知識がなくても着実に進められるよう、スモールステップで解説しています。

---

## 【フェーズ0】事前準備（移管を始める前に）

まずは必要な情報と心の準備をします。ドメインのお引越しには数日〜1週間程度かかります。

* [ ] **Jimdoのログイン情報の確認**: 管理画面に確実にログインできるか確認する。
* [ ] **決済用クレジットカードの準備**: ドメイン移管費用およびGoogle Workspaceの支払いに必要です。
* [ ] **移管先（レジストラ）を決める**: ドメインを今後どこで管理するか決めます。
  * *※おすすめ: 初心者でも情報が多く、サポート体制がある「お名前.com」や「エックスサーバー（Xserverドメイン）」などが無難です。*

---

## 【フェーズ1】Jimdoから「AuthCode（オースコード）」をもらう

AuthCodeとは、ドメインの「お引越し用パスワード」のようなものです。これがないと移管できません。

* [ ] **1.** Jimdoの管理画面にログインする。
* [ ] **2.** メニューから「ドメイン・メール」または「契約情報」を開く。
* [ ] **3.** ドメイン（`carpath.net`）の「解約・外部への移管手続き」を選択する。
* [ ] **4.** 画面の指示に従い移管手続きを進めると、Jimdoから**「AuthCode（英数字の羅列）」**が発行・メール送信されるので、コピーしてメモ帳などに控えておく。

---

## 【フェーズ2】新しい管理会社へ「移管申請」をする

引越し先（お名前.com等）に対し、「このドメインをこっちで管理します」と申請を出します。

* [ ] **1.** 移管先（例: お名前.com）のサイトにアクセスし、「ドメイン移管（お引越し）」のメニューを開く。
* [ ] **2.** 移管したいドメイン（`carpath.net`）を入力する。
* [ ] **3.** 【フェーズ1】で控えた**「AuthCode」**を入力する欄が出るので、正確に貼り付ける。
* [ ] **4.** 移管手数料（通常1,000円〜2,000円程度）をクレジットカードで決済する。
* [ ] **5.** **【重要】承認メールのクリック**
  * 数時間〜数日後に、Jimdo側（または現在のドメイン登録者）から「本当に移管してよいですか？」という英語や日本語のメールが届きます。
  * メール内のURLをクリックし、「承認（Approve）」ボタンを押します。
* [ ] **6.** 移管完了のメールが移管先から届くまで待つ（これでドメインの引越しは完了です！）。

---

## 【フェーズ3】Google Workspaceの申し込み

ドメインの管理が自社に移ったら、いよいよGoogle Workspaceを契約します。

* [ ] **1.** [Google Workspaceの公式サイト] にアクセスし、「利用を開始する」をクリック。
* [ ] **2.** 会社情報（株式会社パルマス、従業員数など）を入力する。
* [ ] **3.** 「すでにお持ちのドメインを使用する」を選択し、`carpath.net` を入力する。
* [ ] **4.** 管理者用のアカウント（例: `info@carpath.net` や `nagao@carpath.net`）を作成し、パスワードを決める。
* [ ] **5.** プラン（一番安い「Business Starter」で十分です）を選択し、支払情報を入力する。

---

## 【フェーズ4】ドメインの所有権証明（TXTレコードの設定）

Googleから「本当にあなたが `carpath.net` の持ち主か証明してください」と求められます。

* [ ] **1.** Google Workspaceの管理画面（Google Admin）にログインし、「ドメインの所有権を証明」ボタンを押す。
* [ ] **2.** Googleから「TXTレコード」という長い文字列（`google-site-verification=...`）が指定されるのでコピーする。
* [ ] **3.** 新しいドメイン管理会社（例: お名前.com）の管理画面を別のタブで開く。
* [ ] **4.** 「DNS設定」または「ネームサーバー設定」というメニューを開く。
* [ ] **5.** レコード追加画面で以下のように入力して保存する。
  * タイプ：`TXT`
  * 値：【先ほどコピーした文字列】
* [ ] **6.** Googleの画面に戻り、「証明」ボタンを押す。（※反映に数分〜1時間ほどかかることがあります）

---

## 【フェーズ5】Gmailで受信するための設定（MXレコードの設定）

「`@carpath.net` 宛のメールは、今後Googleのサーバー（Gmail）に届けてください」という郵便局の転送届のような設定をします。

* [ ] **1.** 引き続き、ドメイン管理会社（お名前.com等）の「DNS設定」画面を開いたままにする。
* [ ] **2.** 新しく以下のレコード（MXレコード）を追加・保存する。
  * タイプ：`MX`
  * 値：`SMTP.GOOGLE.COM` （※Googleの指示画面に複数表示される場合は、指示通りに優先度と値をすべて追加します）
* [ ] **3.** 同時に、なりすまし防止の「SPFレコード」も追加しておく。
  * タイプ：`TXT`
  * 値：`v=spf1 include:_spf.google.com ~all`

---

## 【フェーズ6】最終確認（テスト）

* [ ] **1.** ご自身のスマホ（個人のGmailやLINEヤフーなど）から、作成した `info@carpath.net` 宛にテストメールを送信する。
* [ ] **2.** Google WorkspaceのGmailを開き、メールが届いているか確認する。
* [ ] **3.** 逆に、`info@carpath.net` から自分のスマホ宛にメールを返信し、問題なく届くか確認する。

以上で、ドメインの独立と、セキュアで高機能なGoogle Workspace（Gmail）の導入が完了です！
